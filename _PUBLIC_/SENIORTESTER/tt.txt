P & C

- test plan
  - test items, features tested, features not tested, exit criteria, suspension criteria,
    deliverables, tesing tasks, environmenal needs, responsabilities, training needs, 
    schedule, risks and contingencies, approvals  (estrategias).
    ver http://www.stellman-greene.com/aspm/content/view/39/41/

- master test plan, level test plan


- recursos necesarios: personas, tiempo, utilidades?

- test management: mantener el test process, infraestructuras de test, testware

- risks -> test strategy

- control respeto al plan. 

- feeback: pesronas, herramientas

- el test plan se modifica continuamente, segun el feedback del control.

- exit / completion criteria
   - test coverage of source code
   - customer fullfillment

- test priorities

- test intensity
